## [2026-02-26] agents_list 工具语义陷阱 ⚠️

**Agent：** 大管家 - 马云

**场景：** 用户问"你有什么 skill"和验证多 Agent 配置时

**发现：** 
- `agents_list` 工具返回的是**当前会话可 target 的 Agent allowlist**（用于 `sessions_spawn`）
- 不是已配置的 Agent 列表
- 正确的查看方式是 `openclaw agents list` 命令

**应用：** 
- 以后查询 Agent 配置状态，直接用 `openclaw agents list` 命令
- 不要依赖 `agents_list` 工具判断 Agent 是否配置
- 工具返回的信息可能是片面的，需要多角度验证

## [2026-02-26] 定时任务配置陷阱 ⚠️

**Agent：** 大管家 - 马云

**场景：** 创建百晓生日报巡检定时任务时

**发现：** 
- 飞书渠道的定时任务必须指定 `--to` 参数（具体 chat_id）
- 只配置 `--channel feishu` 会报错：`No delivery target resolved for channel "feishu"`
- 多 API 调用任务需要设置合理超时（300-600 秒）
- 任务执行失败没有主动通知机制

**应用：** 
- 创建定时任务检查清单：
  - [ ] `--agent` 指定正确的 Agent
  - [ ] `--channel` 指定渠道
  - [ ] `--to` 指定具体目标（飞书 chat_id / Telegram chatId）
  - [ ] `--timeout-seconds` 设置合理超时
  - [ ] 创建后执行 `cron list` 验证
- 每天检查 `cron runs` 查看执行历史
- 在日报中加入"定时任务执行状态"部分

## [2026-02-26] 任务执行五步法 ⚠️

**Agent：** 大管家 - 马云

**场景：** 林哥要求创建 OpenClaw 新手入门文章，大管家没有遵循正确流程导致输出为空

**发现：** 
复杂任务必须遵循以下流程，缺一不可：
1. **头脑风暴** - 先用 `brainstorming` 技能梳理方案，与林哥对齐
2. **资源调研** - 让 @百晓生 - 张一鸣 查 Notebook/现有资料，不重复造轮子
3. **方案确认** - 输出大纲/结构，获林哥确认后再执行
4. **执行生成** - 按确认的方案生成内容
5. **验证交付** - 读取生成内容确认有实质内容，再交付链接

**应用：** 
- 以后任何复杂内容创作任务（文章/报告/文档）都按此流程
- 林哥明确指定协作 Agent 时，必须@对应角色参与
- 输出前必须验证，不能让林哥发现"文档没内容"
- 善用 Notebook 等现有知识库，优先查找再创作

## [2026-02-26] npm 安装失败代理兜底
**场景：** 升级 `@openai/codex` 时，`npm install -g` 卡住或失败。  
**问题：** 执行环境网络受限；默认代理 `127.0.0.1:7890` 在沙箱里可能 `EPERM`，去代理后又可能 `ENOTFOUND`。  
**解决：**
1. 先检查 `npm ping` 判断网络类型；
2. 失败时优先尝试代理执行：`withproxy npm install -g <pkg>`；
3. 若仍失败，切换到主机网络权限执行安装。  
**启示：** 以后遇到安装/下载失败，先做“网络诊断 + 代理兜底”，不要直接判定包有问题。

## [2026-02-26] gateway restart 失败自动修复链路
**场景：** `openclaw gateway restart` 因配置错误失败，服务无法拉起。  
**问题：** 过去依赖手工看日志改配置，恢复慢且容易二次误改。  
**解决：** 使用 `openclaw-restart-heal.sh`：
1. 先重启+健康检查；
2. 失败则收集日志上下文并调用 `codex exec` 最小修复；
3. 修复后自动重启+健康检查；
4. 若仍失败，回滚到 guard 备份兜底。  
**启示：** restart 必须有自动恢复闭环，避免人工介入成为单点瓶颈。
