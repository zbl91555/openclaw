## 📚 OpenClaw 社区案例调研

### 1. memU —— 24 小时主动式记忆代理（10.5k 星）
- **项目地址**: https://github.com/NevaMind-AI/memU
- **核心功能**: 为全天候 AI 代理设计的记忆框架
- **关键特性**:
  - 记忆结构像文件系统（文件夹/标签/符号链接）
  - 主动捕捉用户意图，无需命令即可行动
  - 降低长期运行的 LLM token 成本
  - 支持邮件处理、日程管理、市场跟踪等主动任务
- **适用场景**: 个人助理、主动服务代理
- **在线试用**: memu.bot

### 2. ClawRouter —— 智能模型路由器（3.4k 星）
- **项目地址**: https://github.com/BlockRunAI/ClawRouter
- **核心功能**: 本地 LLM 路由层，自动选择最优模型
- **关键特性**:
  - 41+ 模型，覆盖 7 个提供商（OpenAI/Anthropic/Google/DeepSeek/xAI 等）
  - 15 维度评分系统，本地路由，延迟<1ms
  - 四种模式：eco/auto/premium/free
  - USDC 加密货币按次付费，非托管
  - 混合成本 $2.05/M，比 Claude Opus 省 92%
- **适用场景**: 高频使用 OpenClaw，需要成本优化

### 3. Mission Control —— 企业级管理面板（847 星）
- **项目地址**: https://github.com/abhi1693/openclaw-mission-control
- **核心功能**: OpenClaw 团队管理和任务协调系统
- **关键特性**:
  - 多团队代理协作
  - 审批流程管理

  - 活动审计日志

- **适用场景**: 企业/团队部署 OpenClaw

## ⚠️ 教训记录 - TTS 语音格式规范

**问题**: 2026-02-25 发送 TTS 语音时，文本中混入了 markdown 格式（**加粗**）和 emoji（🦞☕️），导致语音引擎直接读出这些字符，听起来很怪。

**正确做法**:
- TTS 文本必须是纯文本
- 不使用任何 markdown 格式（**加粗**、`代码`、- 列表等）
- 不使用 emoji 表情
- 数字、英文、专有名词用中文口吻表达（如 "memu 点 bot"、"1 万 5 千星"）

**错误示例**:
```
**第一个，memU** —— 记忆框架，1 万 5 千星 🦞
```

**正确示例**:
```
第一个，memU，记忆框架，1 万 5 千星。
```

**执行标准**: 以后所有 TTS 语音发送前，必须检查文本是否为纯文本，无格式、无 emoji。
